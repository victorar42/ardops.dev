# Contract: syntax.css budget & structure

**Feature**: 016-syntax-highlighting

---

## Path y propiedad

- **Path**: `assets/css/syntax.css`
- **Generador**: `scripts/build-syntax-css.js`
- **Tema fuente**: `github-dark-default` (Shiki preset)
- **Commiteado**: sí (el repo es la fuente de verdad).
- **Servido desde**: GitHub Pages, mismo origen, `Content-Encoding: gzip` automático.

## Budget

- **Tamaño máximo**: ≤ **5 120 B gzip** (`gzip -c -9 assets/css/syntax.css | wc -c`).
- Verificado por `tests/syntax-css-size.sh` (gate CI).
- Si el budget se excede, la PR no mergea. Mitigaciones aceptables:
  - Recortar clases `.tok-*` no usadas por la allowlist.
  - Mover decoraciones secundarias (italic en comments) a opcional.

## Selectores permitidos

```css
/* Container del bloque */
pre.shiki { /* fondo, padding, border-radius */ }
pre.shiki code { /* font-family, line-height */ }

/* Línea individual */
pre.shiki .line { /* display, white-space */ }

/* Tokens (16 clases) */
.tok-keyword     { color: …; }
.tok-string      { color: …; }
.tok-comment     { color: …; font-style: italic; }
.tok-function    { color: …; }
.tok-number      { color: …; }
.tok-operator    { color: …; }
.tok-variable    { color: …; }
.tok-type        { color: …; }
.tok-constant    { color: …; }
.tok-property    { color: …; }
.tok-punctuation { color: …; }
.tok-tag         { color: …; }
.tok-attribute   { color: …; }
.tok-regex       { color: …; }
.tok-deleted     { color: …; }
.tok-inserted    { color: …; }
```

## Selectores prohibidos

- `*` (universal)
- IDs (`#foo`)
- Atributos (`[style]`, `[data-*]`)
- Pseudoclases interactivas (`:hover`, `:focus`) — no son tokens.
- Media queries — el sitio es dark-only por diseño.
- `@import`, `url()` apuntando a externals.

## Header obligatorio

Las primeras líneas del archivo deben ser:

```css
/* Generated by scripts/build-syntax-css.js
 * Theme: github-dark-default
 * Languages: 21
 * Do not edit by hand — run `node scripts/build-syntax-css.js` to regenerate.
 */
```

Esto hace evidente que el archivo es generado y previene edits
manuales que después se sobreescriban.

## Reproducibilidad

`scripts/build-syntax-css.js --check` debe terminar con exit 0 si y
solo si el archivo en disco coincide byte-a-byte con la regeneración.
Gate CI: `tests/build-syntax-css-check.sh` (opcional; redundante con
`syntax-css-size.sh` si confiamos en revisar manualmente).

## Compatibilidad con `base.css`

- `pre.shiki` debe coexistir con la regla existente que renderiza
  `<pre>` mono plano. La opción más limpia: la regla actual `pre` en
  `assets/css/base.css` (o similar) se mantiene como fallback; las
  reglas con mayor especificidad (`pre.shiki`) sobrescriben fondo y
  padding.
- Cero conflictos de variables: los colores `--bg-secondary`,
  `--accent` siguen siendo source-of-truth; `syntax.css` los referencia
  o define sus propios valores hex consistentes con la paleta.
