Stack & herramientas

Lo que uso día a día como DevSecOps en 2026, y por qué. Sin afiliados, sin patrocinios, sin imágenes: solo el por qué.

Hardware

MacBook Pro 14" (Apple Silicon, 32 GB RAM)
Combo de batería, silencio y rendimiento difícil de igualar para una laptop de trabajo. Compila contenedores y corre VMs sin que el ventilador se entere.
Monitor externo 27" 4K
Espacio real para tener IDE, terminal y navegador en paralelo sin convertir cada PR en un Tetris de ventanas.
Teclado mecánico TKL low-profile
Sin numpad, switches livianos. Menos viaje del dedo al mouse y muñecas más felices al final del día.
Auriculares con cancelación activa
Para entrar en flow profundo cuando la oficina (o la casa) tiene mucho ruido. Imprescindible para sesiones largas de troubleshooting.

Sistema operativo y shell

macOS
Estabilidad de un Unix decente con drivers que funcionan. Para servidores, Linux; para mi diaria, esto.
zsh + starship
Prompt rápido, contextual (rama git, k8s context, cloud profile) y sin sorpresas. Cero ricing innecesario.
Homebrew
Gestor de paquetes que funciona y no se pelea con el sistema. La mayoría de mi stack se instala desde acá.
direnv
Cargar variables de entorno por proyecto sin contaminar el shell global. Imprescindible cuando tocás varios clientes.

Editor / IDE

Visual Studio Code
Buen balance entre velocidad y ecosistema. Lo uso con muy pocas extensiones para que no se vuelva un IDE pesado.
GitHub Copilot
Acelera lo aburrido (boilerplate, parsers, regex) y libera tiempo para el diseño real. Reviso siempre lo que sugiere antes de aceptar.
Vim bindings
Edición sin despegar las manos del teclado. La curva inicial vale cada minuto invertido.

Terminal y CLI

iTerm2
Splits, perfiles por proyecto, búsqueda dentro del buffer. Mejor experiencia que el Terminal por defecto sin volverse pesado.
tmux
Sesiones persistentes para no perder el contexto cuando se cae la red o reinicia el equipo.
fzf + ripgrep + bat
Trío para buscar archivos, contenido y leer código en la terminal. Reemplazan abrir el IDE para el 80% de las consultas rápidas.
jq + yq
Sin estos dos, manipular respuestas de APIs o YAML de Kubernetes en la terminal sería un castigo.
gh (GitHub CLI)
Crear PRs, revisar runs de Actions y triagear issues sin cambiar al navegador. Suma productividad real.

Lenguajes y runtimes

Python
Glue language por excelencia: scripts de automatización, herramientas internas, prototipos rápidos. Ecosistema imbatible.
Go
Para CLIs y servicios donde un binario único, sin runtime, marca la diferencia. Compila rápido y se despliega sin drama.
Bash
Lingua franca de CI/CD. No me encanta, pero está en todos lados y vale conocerlo bien.
Node.js
Para herramientas tipo build scripts, gates y este mismo sitio. La cadena de herramientas web sigue siendo la más madura.

DevOps & infra

GitHub Actions
CI/CD donde ya vive el código. Workflows como código, runners hosted o self-hosted, y un marketplace que evita reinventar ruedas.
Terraform
IaC declarativo, multi-cloud, con estado explícito. Su ecosistema (providers, modules, registry) es inigualable hoy.
Docker / OCI
Empaquetar, distribuir y ejecutar lo mismo localmente y en producción. Imágenes pequeñas, multi-stage builds, sin secretos adentro.
Kubernetes
Cuando la complejidad lo justifica. No por moda. Para todo lo demás, prefiero servicios gestionados.
Helm
Para aplicaciones Kubernetes con configuración por entorno. Charts propios cuando hace falta, oficiales cuando bastan.

Security & compliance

GitHub Advanced Security
Code scanning, secret scanning y dependency review en el mismo PR. Seguridad como parte del flujo, no como gate al final.
Trivy
Escaneo rápido de imágenes, IaC y filesystems. Open source, sin compromisos, fácil de meter en un workflow.
OWASP ZAP
DAST cuando se necesita una mirada activa contra una app corriendo. Útil para baseline scans en CI.
1Password CLI
Secretos fuera del repo, fuera del entorno. Inyectados solo cuando se necesitan, auditados siempre.
SOPS + age
Para secretos que sí viven en repos (configs cifradas). Cifrado por archivo, llaves rotables, integración limpia con git.

Cloud & servicios

AWS
El cloud que más uso por proyectos de banca. Ecosistema enorme; respeto saludable a la factura de fin de mes.
GitHub Pages
Hosting de este sitio: estático, gratis para repos públicos, HTTPS forzado. Suficiente y honesto.
Cloudflare DNS
DNS rápido y con buen panel. (Para `ardops.dev` el DNS vive en GoDaddy por decisión documentada en la constitución.)

Productividad y notas

Obsidian
Notas en Markdown, linked, locales. Sin lock-in: si mañana me voy, mis notas siguen siendo archivos en una carpeta.
Raycast
Reemplaza Spotlight con scripts, snippets y workflows. Lanzador, calculadora, gestor de ventanas, todo en uno.
Linear / GitHub Issues
Tracking ligero. Linear cuando hay equipo y producto; Issues cuando es solo código y PRs.
Calendly
Agendar reuniones sin pasar por el ping-pong de "¿cuándo te queda?". Respeta el tiempo de ambos.

Abandoned

Vagrant
VMs locales reproducibles fueron geniales en su momento. Hoy Docker (y a veces Lima/Colima) cubren el caso con menos overhead.
Jenkins
Lo respeto, pero su modelo XML+plugins se siente del siglo pasado al lado de pipelines como código en GitHub Actions.
Make como build runner para sitios web
Era cómodo, pero los scripts en `package.json` son más portables y todos los entienden sin documentación adicional.
